## Invariants — do not break

- DB prices are CENTS 0..100. PRAGMA user_version=1 guards the one-time
  migration; never multiply again. Polymarket APIs speak 0..1 fractions —
  convert at the door (round(p*100, 2) in poll + backfill). fmtCents takes
  cents. CSV writes cents %.2f.
- Group markets by event_slug, never title (repeat fixtures share titles).
- Chart brush window = timestamps [fromTs, toTs], never indexes; Brush is
  keyed on data identity so it re-seats on new data, not mid-drag.
- Exactly ONE uvicorn worker — the collector runs in-process.
- All timestamps UTC, format 2026-07-16T10:40:23Z (created_at matches ticks).
- Closed market = its tokens absent from /midpoints 3 polls in a row (book
  pulled). Pinned prices (0.05¢/99.95¢) are NOT closed — keep polling.
- Unique index (outcome_id, ts) + INSERT OR IGNORE → retries never duplicate.

## How it works (facts that took effort to learn)

- Backfill: /prices-history gives 1-min fidelity ONLY with explicit
  startTs/endTs windows (~3-day pages). Walk back from created_at; stop when
  a window adds no new points (the API re-serves its earliest data forever).
  Backfill stores only ts < created_at, live only after → disjoint by design.
- Collector sends ONE batched POST /midpoints per poll interval per cycle no
  matter how many markets (~2-3 req/10s vs 500/10s allowed; docs:
  https://docs.polymarket.com/api-reference/rate-limits — they throttle, not
  reject). Backoff + jitter on 429; failed cycles are skip-and-log.
- Search bar routing: ";" = multi-URL lookup, "," = comma screener, else
  single event lookup. Screener prop filter matches QUESTION text, not
  outcome labels ("o/u 2.5", "match o/u", "strikeouts", "toss",
  "total kills"). Sport tag ids: soccer/football 100350, tennis 864,
  baseball 678, mlb 100381, basketball 28, nba 745, nfl 450, cricket 517,
  esports 64.
- client.js request() retries once on message-less 4xx / network drop.
  Reason: a proxy on the client's old plain-HTTP connection injected fake
  400s (server logs show zero 400s ever). HTTPS was the real cure. All
  backend→UI field mapping lives in client.js only.
- Routes: #/ (dashboard; page/per/status/view kept in URL), #/market/{id},
  #/screener (V2 preview). No SPA fallback for clean paths — deliberate,
  needs ~5 backend lines when V2 is approved.
- Dashboard: groups sorted open→paused→closed, newest-added first within
  each band; single-prop events render flat; multi-prop = blue +/− circle
  accordion with tree connector lines (T.series[0]); pagination 10-100
  per page, default 20; scroll-to-top on page change.
- History page: current-price chips + end dots on chart; Live toggle
  (follows the newest edge only while live); manual refresh never moves the
  window; default window = last 10 min for open markets, full history for
  closed; dashed price-level lines 10-50¢, click one → touch dots (max 300).
- CSV filename = slugified market name + date added.
